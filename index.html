TTS 与字幕07 / 12
TTS provider 处理流程

口播文本如何变成
视频时间线

synthesizeVoiceover() 的职责是把口播文本转成三类下游产物：音频文件、字幕片段和 durationMs。火山 V3 主链路整段请求 sentence timing；文本分段逻辑位于本地 MOSS provider 内部。

01 / primary 优先调用火山 TTS

默认走 Volcengine V3 chunked API，整段文本发送给 provider，并请求 subtitle / sentence timing。

02 / timing 解析时间线

优先使用 provider 返回的 sentence timing；如果缺失，再用文本长度估算兜底时长。

03 / local fallback 本地 TTS 兜底分段

当 fallback provider 配置为 MOSS 时，在 provider 内部按强停顿符拆句，过长句再按软停顿符拆分。

04 / timeline 输出时间线

audio、segments、captions、durationMs 落盘，Remotion 用 duration 换算 frames。

Provider 选择主 provider、重试、fallback 都由配置控制。
文件校验渲染前确认音频文件存在，避免生成无声视频。
视频对齐时长决定帧数，字幕片段决定字幕出现位置。
